# プロジェクトナレッジ (PROJECT_KNOWLEDGE.md)

このファイルは、複数のエージェント（または開発者）が並列で作業を進めるための「共有知識（Knowledge）」です。
プロジェクトの文脈、アーキテクチャの決定事項、および作業ルールをここに集約します。

## 1. プロジェクト概要
- **目的**: 飛行機の便名からフライト情報と現地の天気を表示するアプリの商用化。
- **現状**: プロトタイプ完成済み（モックデータ稼働）。商用化に向けた機能拡張フェーズ。

## 2. 技術スタック & アーキテクチャ
- **フレームワーク**: React + Vite
- **言語**: JavaScript (将来的にTypeScript移行推奨)
- **スタイリング**: Vanilla CSS (`src/index.css` にグローバル変数あり。プレミアムダークモード)
- **データ層**: 現在は `src/services/api.js` でモックデータを返却。ここをリアルタイムAPIに置き換えるのが直近の目標。

## 3. 採用予定のAPI (調査結果)
### フライト情報: AviationStack
- **理由**: 無料枠で月100リクエストまで利用可能。
- **要件**: ユーザーからのフィードバックにより、**リアルタイム性（遅延など）よりも、目的地と日程の特定**が主目的であるため、スケジュール情報が正確であれば十分。
- **代替案**: OpenSky Network (無料だがADS-Bデータ中心でステータス情報としては扱いづらい場合がある)。

### 天気予報: OpenWeatherMap
- **理由**: 無料枠で日1,000リクエストまで利用可能。知名度が高く、ドキュメントも豊富。
- **要件**: **正確性が最重要**。旅行中の行動計画に関わるため、信頼性の高いデータソースが必要。
- **代替案**: WeatherAPI.com (予報期間が長い)。

## 4. 並列作業プロトコル (エージェント向け指示書)
複数のエージェントが同時に作業する場合、以下のルールに従ってください。

### 手順
1. **タスクの確認**: ルートディレクトリの `TASKS.md` を読み込む。
2. **タスクの宣言**: 着手するタスクのステータスを `[ ]` から `[/]` (進行中) に書き換える。
   - *衝突防止のため、必ず最初にこれを行うこと。*
3. **実装**: コードを変更する。
4. **完了報告**:
   - `TASKS.md` のステータスを `[x]` (完了) にする。
   - アーキテクチャに変更があった場合は、この `PROJECT_KNOWLEDGE.md` を更新する。

### 推奨される役割分担
- **Agent A (データ担当)**: フェーズ1のAPI実装を担当。`src/services/` を中心に作業。
- **Agent B (テスト担当)**: フェーズ2のテスト環境構築を担当。`src/` のコードには触れず、テストファイルを追加する。
- **Agent C (UI担当)**: フェーズ3のUI改善を担当。`src/components/` や CSS を中心に作業。

## 4. ディレクトリ構造の重要ポイント
- `src/services/api.js`: APIロジックは全てここに集約する（コンポーネントに直書きしない）。
- `src/components/`: 再利用可能なUI部品。
- `TASKS.md`: タスクの進捗管理。
- `ROADMAP.md`: 長期的なゴール。

このファイルを「コンテキスト」として読み込むことで、どのエージェントもプロジェクトの全容を理解して作業を開始できます。
